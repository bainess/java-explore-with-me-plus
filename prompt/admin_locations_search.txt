Описание
Реализовать поиск опубликованных событий, чьи координаты (lat, lon) находятся внутри окружности указанной администратором локации (центр + радиус). Добавить эндпоинт в админскую часть событий.

Состав работ

Добавить в EventRepository метод с нативным SQL-запросом, использующим функцию distance:
@Query(value = 
    "SELECT e.* FROM events e " +
    "JOIN admin_locations l ON l.id = :locId " +
    "WHERE distance(e.lat, e.lon, l.lat, l.lon) <= l.radius " +
    "AND e.state = 'PUBLISHED'",
    countQuery = "SELECT count(*) FROM events e " +
                 "JOIN admin_locations l ON l.id = :locId " +
                 "WHERE distance(e.lat, e.lon, l.lat, l.lon) <= l.radius " +
                 "AND e.state = 'PUBLISHED'",
    nativeQuery = true)
Page<Event> findByLocation(@Param("locId") Long locId, Pageable pageable);
Убедиться, что имена полей соответствуют events.lat, events.lon.

Добавить сигнатуру в интерфейс EventService:
List<EventFullDto> getEventsByLocation(Long locId, int from, int size);
Реализовать в EventServiceImpl:
Проверить существование локации через AdminLocationRepository.findById, иначе бросать NotFoundException.
Сформировать Pageable pageable = PageRequest.of(from/size, size).
Получить Page<Event> через eventRepository.findByLocation(locId, pageable).
Для каждого события получить confirmedRequests (использовать существующий метод getConfirmedRequests) и views (вызов StatsClient, как в getEventPublic).
Вернуть список EventFullDto.
Добавить эндпоинт в EventAdminController (или отдельный контроллер):
GET /admin/locations/{locId}/events?from=0&size=10 → List<EventFullDto> (200).
Если эндпоинт размещается в AdminLocationController, указать соответствующий метод.
Написать интеграционные тесты с предварительно созданными локацией и несколькими событиями (внутри/вне радиуса), проверить точность выборки.
Критерии готовности

Эндпоинт возвращает только опубликованные события, координаты которых находятся на расстоянии ≤ радиуса от центра локации.
Поддерживается пагинация.
В DTO событий заполнены просмотры и количество подтверждённых запросов.
При отсутствии событий возвращается пустой список, при несуществующей локации – 404.
