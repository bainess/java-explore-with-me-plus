Описание
Добавить в schema.sql создание таблицы admin_locations и регистрацию функции distance. Создать JPA-сущность AdminLocation и репозиторий для работы с ней.

Состав работ

Внести в schema.sql DDL:
CREATE TABLE IF NOT EXISTS admin_locations (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name VARCHAR(120) NOT NULL,
    lat FLOAT NOT NULL,
    lon FLOAT NOT NULL,
    radius FLOAT NOT NULL CHECK (radius > 0),
    CONSTRAINT uq_admin_location_name UNIQUE (name)
);
Добавить в schema.sql определение функции distance (полный текст из ТЗ).
Создать класс-сущность AdminLocation в пакете ...location.model с аннотациями @Entity, @Table(name = "admin_locations"), полями согласно таблице.
Создать интерфейс AdminLocationRepository в ...location.dal, расширяющий JpaRepository<AdminLocation, Long>.
Написать простой тест репозитория (сохранение и поиск) с использованием H2/тестовой БД.
Критерии готовности

Приложение запускается, таблица admin_locations создаётся автоматически.
Функция distance доступна в БД (проверяется тестовым запросом).
Сущность и репозиторий корректно сохраняют и читают данные.
