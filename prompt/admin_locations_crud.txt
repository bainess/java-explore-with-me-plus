Описание
Реализовать бизнес-логику управления административными локациями и REST-контроллер для администратора.

Состав работ

Создать LocationServiceImpl в ...location.service, имплементирующий LocationService.
Логика:
Создание: проверка уникальности имени через existsByName → сохранение через AdminLocationRepository → возврат LocationDto.
Обновление: поиск по ID → проверка, что новое имя не конфликтует с существующим (findByNameIgnoreCaseAndIdNot) → обновление полей → сохранение → возврат DTO.
Удаление: проверка существования → удаление.
Получение по ID: поиск, иначе NotFoundException.
Получение всех: PageRequest.of(from/size, size, Sort.by("id")) → маппинг в DTO.
Использовать общие исключения NotFoundException, ConflictException.
Создать AdminLocationController (/admin/locations):
POST / (201)
PATCH /{locId} (200) – опционально проверить длину имени на уровне контроллера, если требуется
DELETE /{locId} (204)
GET / (200)
GET /{locId} (200)
Логировать входящие запросы.
Написать модульные тесты сервиса с моком репозитория и WebMvc-тесты контроллера.
Критерии готовности

Все CRUD-операции работают корректно, валидация на уникальность имени соблюдается.
Пагинация возвращает ожидаемое количество записей.
При ошибках возвращаются корректные HTTP-коды и сообщения об ошибках.
